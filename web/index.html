Données

Données d'origine


Il y a trois sources principales pour nos données d'origine:

1. http://opendata.paris.fr

2. http://www.data.gouv.fr

3. https://www.omdbapi.com

Toutes les donées sur ces sites sont "open sources" et on peut faire le text mining sur ces donées légalement.

Les données sur le site de "OMDB" sont été récupérées en utilisant APIs. Celles sur les autres deux sites sont déjà modélisées en plusieurs formats :

tournagesdefilmsparis2011.xml

monuments.json

Ressource sans nom.ods

Pour le fichier contenant la liste de monuments, on le fait par la main et les noms des monuments sont trouvés sur le site "monumentdeparis"et le fichier momument est liste_de_monuments.txt.


Traitements

A. Les données sur les monuments ( script )

Créer la liste de momuments par la main, ajouter les coordonées et rendre deux fichiers dont l'un est en format xml et l'autre est en geoJson.

- Récupérer les noms des momuments à Paris

- Éliminer par la main des lieux qui ne sont pas intramuros, etc. Par ex. Les Champs Elysée

- Ajouter les coordonnées des monuments en utilisant les API de Open Street Maps et GeoNames

- Transformation en format geoJson

- Transformation en format xml

- Établir la grammaire DTD et relaxng


B. Les données pour les films ( script )

Le fichier d'origine est en format XML et finalement, deux fichiers en format xml et un fichier en format Geojson sont générés.

- Élinimer les noeuds films qui n'ont pas les coordonnées

- Séparer les coordonées en deux noeuds

- Eliminer les lieux de tournage qui ne sont proches à aucun monument ( script )

- Ajouter le nom du monument comme un nouveau noeud du noeud film restant et ajouter "type_radius" comme l'attribut ( script )

- Transformation en Geojson ( script )


C. Les données sur le genre et le pays d'origine du film ( script )

Le fichier sortie est en format xml

- Faire un programme pour récupérer le genre et le pays d'origine du film qui existe dans notre fichier de film en utilisant APIs
